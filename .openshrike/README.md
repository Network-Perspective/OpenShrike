# OpenShrike Init

This directory is generated by `shrike init`.

- `checks/` stores the project-local Markdown checks that Shrike executes.
- `project.json` stores repo-local Shrike defaults such as the selected policy and scan settings.
- `opencode.json` is a Shrike-owned OpenCode overlay for read-only scans.
- `.gitignore` keeps generated `artifacts/` files out of version control.
- User-global OpenCode config and credentials remain outside this repository.

Re-run `shrike init` to seed additional checks from a different policy or choose a different model/runtime.
After initialization, `shrike scan` reads only Markdown checks from `checks/`.
